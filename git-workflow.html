Introdução

O objetivo dessa página é trazer informações sobre o Git e o GitHub assim como seus comandos principais.

Git:

O Git é um sistema de controle de versão
distribuído amplamente utilizado para o rastreamento de mudanças em projetos de software.
O Git é uma ferramenta poderosa e versátil, essencial para desenvolvedores que desejam controlar e colaborar eficientemente no desenvolvimento de software.

GitHub:

O GitHub é uma plataforma de hospedagem de
código-fonte que utiliza o sistema de controle de versão Git.
O GitHub se tornou uma parte essencial do ecossistema de desenvolvimento de software, facilitando a colaboração, o controle de versão e a comunicação entre
desenvolvedores em todo o mundo.

Informações

Clone

O comando git clone é usado para criar uma cópia local de um repositório Git
existente. Isso é útil para obter uma versão do código-fonte em seu próprio ambiente de desenvolvimento.

Exemplo: git clone https://github.com/exemplo/um-repositorio.git

Branch

Um branch no Git é uma ramificação independente do desenvolvimento,
permitindo que você trabalhe em funcionalidades ou correções de bugs sem afetar o código principal.

Exemplo: Criar um branch: git branch nome-do-branch
Mudar para um branch: git checkout nome-do-branch

Commit

Um commit é uma instância de mudança no repositório.
Ele representa uma versão específica do código, acompanhada de uma mensagem
que descreve as alterações realizadas.

Exemplo: git commit -m "Mensagem descritiva"

Merge

O merge é o processo de combinar as alterações
de dois branches diferentes. Isso é frequentemente usado
para incorporar o trabalho de uma branch de
desenvolvimento de volta à branch principal.

Exemplo: git merge branch-a-ser-mergeada

Pull Request

Um pull request é uma solicitação feita
por um desenvolvedor para incorporar suas alterações de um
branch em outro. Isso permite revisão de código e
discussão antes de mesclar as alterações.

Pull

O comando git pull é usado para buscar e mesclar as alterações de um repositório remoto para o repositório local.

Exemplo: git pull origin nome-do-branch
